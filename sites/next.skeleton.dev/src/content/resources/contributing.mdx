## Astro

The Skeleton documentation uses [Astro](https://astro.build/). View the following integrations in `astro.config.mjs`:

- [Tailwind](https://docs.astro.build/en/guides/integrations-guide/tailwind/)
- [React](https://docs.astro.build/en/guides/integrations-guide/react/)
- [Svelte](https://docs.astro.build/en/guides/integrations-guide/svelte/)
- [MDX Markdown](https://docs.astro.build/en/guides/markdown-content/)
- [Prettier](https://docs.astro.build/en/editor-setup/#prettier)
- [Expressive Code](https://expressive-code.com/)
- [Astro-Icon](https://github.com/natemoo-re/astro-icon)
- [Astro-Auto-Import](https://github.com/delucis/astro-auto-import/tree/main/packages/astro-auto-import)
- [Nanostores](https://docs.astro.build/en/recipes/sharing-state-islands/#why-nano-stores) | [Persistent](https://github.com/nanostores/persistent)

## App Structure

The entire app structure is houses within the `/src` directory, including the following subdirectories:

| Path          | Description                                |
|---------------|--------------------------------------------|
| `/components` | Contains the doc site component.           |
| `/content`    | Content collections for MDX page content.  |
| `/examples`   | Documentation page code block examples.    |
| `/icons`      | Contains doc site custom SVG icons.        |
| `/layouts`    | The Astro page layouts.                    |
| `/pages`      | The Astro page route templates.            |
| `/styles`     | All doc site stylesheets.                  |

## Adding MDX Pages

1. Browse to `/content` and create a new `.mdx` file within the appropriate content collection.
2. Be sure to complete all required Frontmatter metadata within the `---` brackets.
3. The page will be automatically added to the sidebar navigation.

## General Usage

### MDX Markdown

View the [Astro MDX Documention](https://docs.astro.build/en/guides/markdown-content/) for full usage, or see `/content/resources/_markdown.md` for examples.

> TIP: Skeleton styles are automatically applied via the `components` export. This must be included on every MDX page.

```astro
---
# (frontmatter)
---

export const components = componentSet {/* <-- ADD THIS! */}

{/* (content) */}
```

### Icons

Icons are provided via [Astro-Icon](https://github.com/natemoo-re/astro-icon) and [Iconify](https://iconify.design/). Both [HeroIcons Outlined](https://icon-sets.iconify.design/heroicons/) and [Simple Icons](https://icon-sets.iconify.design/simple-icons/) are available.

### Linting

See the [Astro Prettier](https://docs.astro.build/en/editor-setup/#prettier) documentation for usage details.

## Global Components

A selection of globally accessible uitlity components provided to MDX contents via `astro-auto-importer`.

> IMPORTANT: These are globally registered and do not need a local import per page.
>
> See `astro.config.mjs` > `AutoImport()` for details.

### Essential Code Blocks

Please refer to the [Expressive Code](https://expressive-code.com/) documentation for details. You can use [Markdown fences](https://expressive-code.com/key-features/syntax-highlighting/) or the full component.

```html
<Code code={AddCodeHere} lang="html" />
```

### Previewer

Allows you to quickly tab between a rendered component and the source code.

```ts
import Example from "@examples/foo/Example.astro";
import ExampleRaw from "@examples/foo/Example.astro?raw";
```

```html
<Preview client:load>
    <Fragment slot="preview"><Example /></Fragment>
    <Fragment slot="code"><Code code={ExampleRaw} lang="html" /></Fragment>
</Preview>
```

## Tables

Use [Markdown tables](https://www.tablesgenerator.com/markdown_tables), or see the custom doc Table component in `/src/components/docs/Table.astro`. The Table component will auto-generate headings based on the key names in each object row.

```ts
const tableProps: { name: 'autocollapse', type: 'boolean', value: 'false', description: 'Set the auto-collapse mode.' }
```

```html
<Table label="Props" data={tableData} tonal />
```

> TIP: You can optionally use either `tonal` or `kbd` props to highlight content in the first column.
