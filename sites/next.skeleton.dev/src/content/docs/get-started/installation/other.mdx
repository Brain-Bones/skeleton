The core of Skeleton is our robust [Tailwind Plugin](/docs/get-started/plugin), which is framework agnostic. This provides access to all [design system](/docs/get-started/fundamentals#design-system) and [Tailwind-based](/docs/get-started/fundamentals#tailwind) features - everything *except* components. The only prerequisites are as follows.

- Node-based package management (NPM, PNPM, etc)
- Tailwind CSS

The exact instructions for installing Skeleton will differ per framework, but we've provided a generic guide below.

<Process>
    <ProcessStep step="1">
        ## Create a Project
        Create any web-based application of your choice.
    </ProcessStep>
    <ProcessStep step="2">
        ## Install Tailwind
		Follow the [official instructions](https://tailwindcss.com/docs/installation/framework-guides) to install Tailwind for your application of choice.
    </ProcessStep>
    <ProcessStep step="3">
        ## Install Skeleton
        Install the Skeleton core package to gain access to the Tailwind plugin.
        ```console
        npm i -D @skeletonlabs/skeleton@next
        ```
    </ProcessStep>
    <ProcessStep step="4">
        ## Configure Tailwind
        Open `tailwind.config` in the root of your project. Note the exact structure may differ slightly for your project.
        ```ts title="tailwind.config" {1,2, 13-16}
		import { skeleton } from '@skeletonlabs/skeleton/plugin';
		import * as themes from '@skeletonlabs/skeleton/themes';

        /** @type {import('tailwindcss').Config} */
		export default {
			content: [
				// NOTE: preserve any settings provided here
			],
			theme: {
				extend: {},
			},
			plugins: [
				skeleton({
					// NOTE: each theme included will be added to your CSS bundle
					themes: [themes.cerberus, themes.rose],
				}),
			],
		};
        ```
    </ProcessStep>
	<ProcessStep step="6">
        ## Set Active Theme
        Open the file containing the `<body>` tag for your project and set the active theme using the `data-theme` attribute.
        ```html
        <body data-theme="cerberus">...</body>
        ```
    </ProcessStep>
    <ProcessStep step="check">
        ## Run the Project
        Start the dev server for your framework of choice.
    </ProcessStep>
</Process>
