There’s no single ‘correct’ way to create wireframes. The goal is to give form to your concepts, test them, and gather feedback or collaborate with others. 

Using tools like Figma, wireframes let you visualize strengths and weaknesses, making it easy to refine your design. At this stage, the only real constraint is screen size. In this guide you will learn how to create a wireframe based on a `Profile section` example.

        ### Create Screen container
The best practice is to start from a mobile screen because it will be much easier to adapt the UI to a bigger screen later than vice versa. Mobile devices lead global web traffic at around 58%, so it makes sense to consider mobile devices' experience first.

In our case, it will be the `Desktop screen` because we can best demonstrate the concept of creating wireframes there.

Insert the `Screen template`¹ component into the canvas
        <Image src={ImgAddScreen} class="rounded-container" alt="Add screen" />
         <p className='pt-4'>On the right side panel, change the `screen size`² property. For our case, we will change it to `laptop / xl / 1280`³.</p>
       <Image src={ImgSelectSize} class="rounded-container" alt="Selecting desktop screen size" />
        <p className='pt-4'>Follow the `Detach me instructions` to be able to place elements inside of the screen template.</p>
        <Image src={ImgDesktopScreen} class="rounded-container" alt="Desktop screen" />
    </ProcessStep>

    <ProcessStep step="3">
    ### Create Modal 

    In this example we embed the ‘Profile section’ into a `modal` because .... But you could also do it without a modal depending on your use case.

As a first step we create the `modal overlay` which acts as a transparent background while the modal is open.
For that, create a rectangle⁴ and fit it into the container at its full width (1280) and height (812). 
      <Image src={ImgRectangle} class="rounded-container" alt="Rectangle" />
         <p className='pt-4'>Apply surface-950 color token⁵ and apply 50% opacity⁶ to ‘modal overlay’</p>
       <Image src={ImgOverlay} class="rounded-container" alt="Overlay" />

        <p className='pt-4'>Now we create the `modal` on top of the `modal overlay`. 
For that, create a rectangle⁷ and apply surface-50 color token⁸.</p>
 <Image src={ImgModal} class="rounded-container" alt="Modal" />
    </ProcessStep>

    <ProcessStep step="4">
    ### Modal headline

    Each page, drawer, or modal should have a heading to provide clear context, help users quickly understand the purpose of the content, and make navigation intuitive.

Create a `Profile`⁹ text and apply `Headline 4`¹⁰ as the text style.
<Image src={ImgHeadline} class="rounded-container" alt="Create headline" />
   
    </ProcessStep>

     <ProcessStep step="5">
    ### Close button

    Modals usually have a `Close` button in the top right corner.

Add the `li:x`¹¹ component to the top right corner of the modal.
`li:` stands for lucid icons. By typing `li:` at the beginning of Assets, you can search specifically for icons.
<Image src={ImgCloseButton} class="rounded-container" alt="Close button" />
   
    </ProcessStep>

   Insert the `Avatar`¹² component and place it below the `Modal headline`.

    ### Add Input

  Insert the `Input`¹³ component below the `Avatar`.
