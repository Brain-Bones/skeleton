Supports <u>all</u> standard Tailwind color utility classes using the following pattern.

```
{property}-{color}-{shade}
```

| Key | Accepted Values |
|---|---|
| Property | `accent`, `bg`, `border`, `caret`, `decoration`, `divide`, `fill`, `outline`, `ring`, `shadow`, `stroke`, `text`
| Color | `primary`, `secondary`, `tertiary`, `success`, `warning`, `error`, `surface` |
| Shade | `50`, `100`, `200`, `300`, `400`, `500`, `600`, `700`, `800`, `900`, `950` |

```html
<div class="bg-primary-500">...</div>
<div class="border border-secondary-600">...</div>
<svg class="fill-surface-950">...</svg>
```

## Contrast Colors

Skeleton themes also include contrast color values for every shade. Use these to set contrasting values for overlapping text and icon fill styles. These are defined via each respective [theme property](/docs/get-started/plugin#colors).

```
{property}-{color}-contrast-{shade}
```

## Pairings

Color Pairings are utility classes that provide an opinionated pairing of colors for light and dark mode.

```
{property}-{color}-{lightModeShade}-{darkModeShade}
```

<ExamplePairings />

Color Parings can be used to generate a heierarchy of visual layers, ranging from foreground to background elements. This uses the same color ramp, but inverts the order from light to dark mode. See this visualized below.

<ExamplePairingsStack />

* We can use shade `950` for light mode and `50` from dark mode respectively for our text color.
* Then use shade `50` from light mode and `950` from dark mode respectively for our app background.
* While using the static `500` shade key brand elements, such as buttons or banners.
* Then reserve multiple layers between for elements such as cards, inputs, and more.

```html
<div class="bg-primary-950-50">Foreground</div>
<div class="bg-primary-900-100">...</div>
<div class="bg-primary-800-200">...</div>
<div class="bg-primary-700-300">...</div>
<div class="bg-primary-600-400">...</div>
<div class="bg-primary-500">Branding</div>
<div class="bg-primary-400-600">...</div>
<div class="bg-primary-300-700">...</div>
<div class="bg-primary-200-800">...</div>
<div class="bg-primary-100-900"></div>
<div class="bg-primary-50-950">Background</div>
```
